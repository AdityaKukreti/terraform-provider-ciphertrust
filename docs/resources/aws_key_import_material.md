# Resource "ciphertrust_aws_key_import_material" (terraform-provider-ciphertrust)

Use this resource to import or re-import key material to an AWS EXTERNAL key.

## Example Usage

```terraform
# Create a CipherTrust AES key
resource "ciphertrust_cm_key" "new_key_material" {
  name      = "new-material"
  algorithm = "AES"
}

# Import new material to the AWS key
resource "ciphertrust_aws_key_import_material" "import_new_material" {
  key_id = "aws-key-id"
  import_key_material {
    import_type           = "NEW_KEY_MATERIAL"
    source_key_identifier = ciphertrust_cm_key.new_key_material.id
    source_key_tier       = "local"
  }
}

# The key material is now in PENDING_ROTATION state and can be made to be the CURRENT key material using a ciphertrust_aws_key_rotation resource.
resource "ciphertrust_aws_key_rotation" "rotate_imported_material" {
  key_id = ciphertrust_aws_key_import_material.import_new_material.key_id
}
```

## Import

This resource does not support import.

`ciphertrust_aws_key_import_material` represents a one-time key material import
operation applied to an existing EXTERNAL AWS key. The operation's input
parameters (source key identifier, import type, key material description) are
write-only and are not returned by the API. There is no way to reconstruct them
from the key's current state, so importing this resource would leave all import
parameters absent from state and cause permanent plan diffs.

To bring an existing key and its imported material under Terraform management,
use the `import_key_material` block within the `ciphertrust_aws_key` resource
and manage the key and its material together as a single resource.
