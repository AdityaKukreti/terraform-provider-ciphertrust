# Resource "ciphertrust_oci_vault" (terraform-provider-ciphertrust)

Use this resource to create and manage OCI vaults in CipherTrust Manager.

## Example Usage

```terraform
resource "ciphertrust_oci_vault" "vault" {
  # Required parameters
  connection_id    = ciphertrust_oci_connection.connection.name
  region           = "oci-region"
  vault_id         = "vault-ocid"
  # Optional parameters
  bucket_name      = "bucket-name"
  bucket_namespace = "bucket-namespace"
}
```

## Import

An OCI vault can be imported using its CipherTrust Manager vault ID. The vault
ID is returned in the API response body as the `id` field and is also visible
in the CCKM interface.

### Important: Behaviour after import

After import, Terraform will attempt to make the vault match your resource
configuration.

If updateable attributes are missing or incorrect they may be updated or
removed during `terraform apply`.

**`connection_id`** - after import, the API returns the **connection name**
(not the UUID) in the `connection` field. Your resource configuration must
use the **connection name** for `connection_id` when managing an imported
vault. Using the connection UUID instead will cause Terraform to plan an
update that changes the connection on the vault.

**`vault_id`** - the OCI OCID of the vault. Returned by the API after import
and must match the value in your configuration.

**`region`** - the vault's region. Returned by the API after import and must
match the value in your configuration.

**`bucket_name` and `bucket_namespace`** - optional fields used for key
backups on HSM-protected Virtual Private Vaults. These are returned by the
API if set. If you do not need them, you can omit them from your
configuration.

To avoid unintended changes, do one of the following:

**Option 1 - Configure the resource block to match the vault's current state**
before running `terraform apply`. Set `connection_id` (using the connection
name), `vault_id`, `region`, and optionally `bucket_name` and
`bucket_namespace` to their current values so that Terraform detects no diff.

**Option 2 - Use `lifecycle { ignore_changes }` to suppress diffs** on
attributes you are not yet ready for Terraform to manage:

```terraform
lifecycle {
  ignore_changes = [
    bucket_name,
    bucket_namespace,
    connection_id,
  ]
}
```

The recommended approach (temporary use of `ignore_changes`):

1. Add the `lifecycle { ignore_changes }` block to the resource
2. Run `terraform apply` (or use `terraform import` for Terraform versions earlier than v1.5.0)
3. Run `terraform state show ciphertrust_oci_vault.example` to inspect the imported values
4. Update your resource configuration to match the imported state
5. Remove `ignore_changes` so Terraform manages the attributes going forward

You can import a resource into Terraform in two ways.

### 1. Import using an import block and terraform apply

Only available for Terraform v1.5.0 and later.

When Terraform processes the `import` block it:

1. Reads the vault from the API using the provided ID and stores the current values as state
2. Compares that state to your resource block configuration
3. Applies any differences as updates to the vault (immediately during the same apply)

All three steps happen within a single `terraform apply` run.

**Step 1 - Import with `ignore_changes`** (safe starting point):

```terraform
import {
  to = ciphertrust_oci_vault.example
  id = "abc12345-1234-1234-1234-123456789012"
}

resource "ciphertrust_oci_vault" "example" {
  connection_id = "<connection-name>"
  region        = "<oci-region>"
  vault_id      = "<vault-ocid>"
  lifecycle {
    ignore_changes = [
      bucket_name,
      bucket_namespace,
      connection_id,
    ]
  }
}
```

**Step 2 - Final config after inspecting state** (remove `ignore_changes`
once values are correct):

```terraform
resource "ciphertrust_oci_vault" "example" {
  connection_id    = "<connection-name>"   # use connection name, not UUID
  region           = "<oci-region>"
  vault_id         = "<vault-ocid>"
  bucket_name      = "<bucket-name>"       # omit if key backups are not used
  bucket_namespace = "<bucket-namespace>"  # omit if key backups are not used
}
```

The example above shows all updatable attributes. Include only the attributes
that apply to your vault - for example, omit `bucket_name` and
`bucket_namespace` if the vault does not use key backups.

### 2. Import using the terraform import command

When `terraform import` is run it:

1. Reads the vault from the API using the provided ID and stores the current values as state
2. Writes the imported state to the state file - no plan or apply is performed

Before running `terraform apply`, apply Option 1 or Option 2 above to avoid
unintended changes. Run `terraform plan` first to review exactly what would be
applied.

```shell
terraform import ciphertrust_oci_vault.example abc12345-1234-1234-1234-123456789012
```
